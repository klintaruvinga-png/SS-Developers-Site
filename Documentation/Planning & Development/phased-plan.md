# SS Developers Website — Phased Plan

## Phase 1: Discovery & Launch Scope
- Confirm final sitemap and launch pages.
- Choose the deployment host and domain strategy.
- Review `plan.md` to extract page structure, positioning, and tone.
- Define the visual direction, layout wireframes, and any required diagrams.
- Identify the minimum viable launch content: Home, Projects, Architecture, How I Work, Work With Me, Contact, Now.
- Confirm Engineering Notes is a fast-follow section, not launch-blocking.

## Phase 2: Foundation & Structure
- Create the Astro project scaffold and configure site settings.
- Implement a shared layout with header, footer, and responsive navigation.
- Set up route structure and page templates for all launch pages.
- Add SEO metadata defaults and accessible page structure.

## Phase 3: Page Buildout
- Build the Home page with hero, dual CTA, capability strip, and featured work.
- Build the Projects overview and link it to both case studies.
- Build the SMC SuperFIB case study page with problem, architecture, challenges, status, and safe visuals.
- Build the Stokvel Society case study page with use case, solution, status, and contrast to SuperFIB.
- Build the Architecture page with system flow diagrams and topology explanation.
- Build the How I Work page with process, AI workflow, and solo-engineer narrative.
- Build the Work With Me page with ideal engagement fit, not-fit guidance, and contact path.
- Build the Contact page with a simple lead capture method.
- Build the Now page with current focus and hardening progress.

## Phase 4: Content, Copy & Safety
- Review all page copy for first-person, engineering-led tone.
- Ensure the lead path is visible but not aggressive.
- Develop the SuperFIB parity story with actual systems engineering detail.
- Develop the Stokvel Society story with trust and fintech framing.
- Review diagrams for proprietary-safety: flow only, no formulas, no sensitive values.
- Review any public repo links or screenshots before publishing.

## Phase 5: QA & Launch Prep
- Run performance checks and Lighthouse audits.
- Validate responsive layout on desktop and mobile breakpoints.
- Test accessibility basics: headings, labels, alt text, keyboard navigation.
- Validate all CTAs and page links.
- Confirm Engineering Notes remains hidden until posts are ready.
- Deploy a staging preview and verify live rendering.
- Confirm the domain and host configuration.

## Phase 6: Launch & Fast-Follow
- Publish the site at the chosen domain.
- Verify the live site matches the planned pages and messaging.
- Confirm contact and lead capture are working.
- Track launch completion.
- Prepare 2-3 Engineering Notes posts for follow-up activation.
- Add Engineering Notes once ready and keep it off launch nav until then.
- Optionally add repo or public links later after proprietary review.
